代理开关 ProxySwitch v3.0 — 使用说明
═══════════════════════════════════════════

■ 简介

  代理开关是一款 Chrome / Edge 浏览器扩展，用于便捷切换代理模式。
  内置 Clash.Meta 后端 + 2 条线路 + 实时下载最新配置，开箱即用。
  支持三种代理模式：直连、规则代理（PAC）、全局代理。


■ 快速开始

  1. 双击运行 setup.bat（首次启动代理后端 + 创建开机自启）

  2. 加载扩展：
     Chrome → chrome://extensions → 开发者模式 → 加载已解压的扩展 → 选择本目录
     Edge   → edge://extensions → 开发人员模式 → 加载解压缩的扩展 → 选择本目录

  3. 点击浏览器工具栏的扩展图标 → 打开总开关 → 选择模式即可翻墙

  以后每次开机自动启动，无需再运行 setup.bat。


■ 界面说明

  ┌─────────────────────────────┐
  │  代理开关            [====] │  ← iOS 风格总开关（默认关闭）
  ├─────────────────────────────┤
  │  [ 直连 | 规则 | 全局 ]     │  ← 三段模式选择
  ├─────────────────────────────┤
  │  ● 连接正常 - Paris, France │  ← 实时地区 + 连通状态
  ├─────────────────────────────┤
  │  ○ 线路 1 - Hysteria2      │  ← 当前线路高亮蓝框
  │     Paris, France | IP:端口 │     实时显示地区与服务器
  │  ○ 线路 2 - Hysteria        │
  ├─────────────────────────────┤
  │  [刷新检测] [✓]自动选择    │
  ├─────────────────────────────┤
  │  仅控制浏览器代理           │
  └─────────────────────────────┘

  三种模式：
    · 直连 — 不使用代理
    · 规则 — GFW 域名走代理、国内直连（推荐日常使用）
    · 全局 — 所有流量走代理

  状态灯：
    · 绿色 = 连接正常
    · 红色 = 后端无响应，请切换线路或点刷新
    · 黄色脉冲 = 正在切换线路 / 检测中
    · 灰色 = 代理关闭


■ 线路系统

  扩展内置 2 条线路，每条对应不同的远端服务器：

    线路 1 — Hysteria2 协议
    线路 2 — Hysteria 协议

  切换线路时自动从 GitLab 下载最新配置，确保 IP 永不过期。
  切换完成后自动查询服务器归属地并显示（如 Paris, France）。

  如果当前线路不通：
    1. 点击另一条线路（2 秒内自动切换）
    2. 或点击「刷新检测」→ 勾选「自动选择」→ 自动切到可用线路


■ 安装 / 卸载

  安装：双击 setup.bat
    · 自动启动 Clash.Meta 代理后端（端口 7890）
    · 自动创建开机自启快捷方式
    · 显示扩展加载指引

  卸载：双击 uninstall.bat
    · 移除开机自启快捷方式
    · 停止 Clash 进程
    · 提示从浏览器移除扩展
    · 可选删除整个目录


■ 常见问题

  Q: 打开扩展提示"未检测"或"无响应"？
  A: 1) 确认 Clash 后端在运行（任务管理器搜 clash）
     2) 点击「刷新检测」
     3) 切换另一条线路试试
     4) 重启电脑后代理会自动启动

  Q: 某网站打不开（规则模式下）？
  A: 该网站可能不在 GFW 列表中，临时切到「全局模式」即可。

  Q: 可以添加更多线路吗？
  A: 线路配置由上游维护（GitLab Alvin9999/new-pac），
     如需更多线路可编辑 clash-helper.ps1 中的节点数量限制。

  Q: 分享给别人怎么用？
  A: 将整个 ProxyToggle 文件夹打包发送，对方解压后
     双击 setup.bat → 加载扩展 → 完成。


■ 技术说明

  · Chrome Extension Manifest V3
  · 规则代理使用内联 PAC 脚本（6069 GFW 域名，O(1) 后缀匹配）
  · clash-helper.ps1 提供本地 HTTP API (127.0.0.1:9877)
  · 切换线路自动从 GitLab 实时下载最新配置 → 重启 Clash → 查询归属地
  · 所有数据仅存储在浏览器本地，不上传


■ 文件清单

  manifest.json         — 扩展清单
  background.js         — Service Worker 核心
  popup.html/css/js     — 弹出界面
  clash-helper.ps1      — PowerShell 后台服务（HTTP API + Clash 管理）
  start-proxy.vbs       — 静默启动 helper
  setup.bat             — 安装向导
  uninstall.bat         — 卸载向导
  clash/                — Clash.Meta 后端 + 配置文件
  rules/gfw-domains.js  — GFW 域名列表（6069 个）
  nodes/                — 节点配置
  icons/                — 扩展图标
  README.txt            — 本文件
